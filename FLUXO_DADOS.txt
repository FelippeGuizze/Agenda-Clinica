## ESTRUTURA DO BANCO DE DADOS

### Tabela: usuarios (Principal)
- id: BIGINT PRIMARY KEY
- nome: VARCHAR(255) NOT NULL
- email: VARCHAR(255) NOT NULL UNIQUE ← VERIFICAÇÃO PRINCIPAL
- senha: VARCHAR(255) NOT NULL
- categoria: INT (0=Admin, 1=Paciente, 2=Profissional)
- paciente_id: BIGINT UNIQUE NULL ← Link para pacientes (se categoria=1)
- profissional_id: BIGINT UNIQUE NULL ← Link para profissionais (se categoria=2)

### Tabela: pacientes
- id: BIGINT PRIMARY KEY
- nome: VARCHAR(255)
- contato: VARCHAR(20)

### Tabela: profissionais
- id: BIGINT PRIMARY KEY
- nome: VARCHAR(255)
- categoria: VARCHAR(100)
- especialidade: VARCHAR(100)

### Tabelas: atendimentos e observacoes_atendimento
- SEM LINKS POR ENQUANTO (como solicitado)

===============================================
## FLUXO DE CADASTRO

1. Usuário preenche formulário:
   - Nome
   - Email ← VERIFICADO SE JÁ EXISTE
   - Senha
   - Tipo (Paciente=1 ou Profissional=2)

2. Se PACIENTE (categoria=1):
   - Cria registro em tabela: pacientes
   - Salva ID do paciente em: usuarios.paciente_id
   - Deixa usuarios.profissional_id = NULL

3. Se PROFISSIONAL (categoria=2):
   - Cria registro em tabela: profissionais
   - Salva ID do profissional em: usuarios.profissional_id
   - Deixa usuarios.paciente_id = NULL

4. Resultado:
   - Tabela usuarios sempre completa (todos os usuários)
   - Tabela pacientes link para usuários categoria 1
   - Tabela profissionais link para usuários categoria 2

===============================================
## SERVIÇOS IMPLEMENTADOS

### SeedService
- seedDatabase(): Insere admin se não existir
- emailJaExiste(email): Verifica se email já está cadastrado

### AppInitListener
- Executa SeedService ao iniciar a aplicação
- Cria automaticamente usuário admin: admin@agenda.com / admin123

### CadastroServlet
- Verifica se email já existe (feedback vermelho se existir)
- Se não existe: cria usuário + paciente/profissional
- Feedback VERDE: cadastro com sucesso
- Feedback VERMELHO: email já existe ou erro

### VerificaEmailServlet
- Endpoint para verificação em tempo real (JSON)
- Retorna: {existe: bool, mensagem: string, cor: string}

===============================================
## MENSAGENS DE FEEDBACK

VERDE (✓ Sucesso):
- Email disponível
- Cadastro realizado
- Login bem-sucedido

VERMELHA (❌ Erro):
- Email já cadastrado
- Dados inválidos
- Erro de conexão
